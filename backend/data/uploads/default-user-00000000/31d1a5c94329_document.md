# TransactionOCR_2 项目详细分析报告

## 项目概述

**TransactionOCR_2** 是一个专业的银行交易流水 OCR 智能体服务，核心功能是将 PDF 或图片格式的银行流水账单通过视觉语言模型（VLM）自动识别提取，并转换为结构化的标准 CSV 数据。

该项目采用 **FastAPI + LangGraph** 的现代微服务架构，通过阿里云百炼 Qwen 系列模型实现高精度的多模态信息提取，同时支持 Dify 平台的动态提示词管理，具备企业级生产部署能力。

### 核心技术栈

| 层级 | 技术选型 | 作用 |
|------|---------|------|
| Web 框架 | FastAPI | 高性能异步 API 服务 |
| AI 编排 | LangGraph | 多节点并行 Agent 工作流 |
| 视觉模型 | Qwen-VL (阿里云百炼) | PDF/图片 OCR 识别 |
| PDF 处理 | PyMuPDF + Pillow | PDF 转图片、压缩优化 |
| 数据处理 | Pandas | 结构化数据处理与导出 |
| 提示词管理 | Dify Workflow | 动态提示词拉取与缓存 |

### 核心能力

- **多格式支持** — 支持 PDF、图片（PNG/JPEG）等多种银行流水文档格式
- **智能识别** — 基于视觉语言模型，自动理解复杂表格结构和多银行格式
- **字段标准化** — 自动映射到 23 个标准字段（流水号、交易日期、对方账户、金额等）
- **余额校验** — 内置余额连续性校验，自动修正借贷方向错误
- **异步处理** — 支持大文件分块并发处理，完成后回调通知
- **动态提示词** — 支持 Dify 平台热更新提示词，无需重新部署

## 二、核心功能模块详解

### 2.1 OCR Agent 工作流（LangGraph）

采用经典的 **Fan-out/Fan-in 并行模式**，两个 LLM 节点同时调用，充分利用模型推理时间重叠：

```
START
  ├── LLM 1: extract_transactions  →  从图片提取交易流水 Markdown 表格
  └── LLM 2: extract_metadata      →  从表头区域提取客户元数据 JSON
         ↓
      merge_results               →  合并流水与元数据为最终 JSON
         ↓
       END
```

**关键特性**：

- **并行执行**：两个 LLM 节点同时调用，充分利用模型推理时间重叠
- **多图支持**：支持一次性输入多张图片，按顺序输出交易记录
- **表头感知**：非首块携带第一页作为表头格式参考（`header_only_first`）
- **元数据跳过**：分块场景下非首块自动跳过元数据提取（`skip_metadata`），避免重复

### 2.2 PDF 分块并发处理

当 PDF 页数超过 `CHUNK_SIZE`（默认 8 页）时，系统自动：

1. **智能分块**：按 CHUNK_SIZE 将页面分组，非首块前插入第一页作为表头参考
2. **并发控制**：`MAX_CHUNK_WORKERS`（默认 6）限制同时发往 LLM 的块数
3. **指纹去重**：基于 `(trans_date, trans_time, trans_amt, account_balance)` 去除重叠行
4. **元数据传播**：从 chunk-0 提取的 name/account_no/account_bank 填充其他块的空值

**重试与二分策略**：

- 每次失败自动重试 2 次（可配置）
- 行数过少时触发系统性截断检测
- 自动二分拆小后递归处理（最多 2 层）

### 2.3 余额连续性校验（Balance Corrector）

这是本项目最具技术深度的模块之一，通过**财务逻辑一致性校验**自动修正 OCR 错误：

**升序校验（时间从早到晚）**：

```
bal[i] = bal[i-1] + delta[i]
其中 delta[i] = +amt（贷方） 或 -amt（借方）
```

**降序校验（时间从晚到早）**：

```
bal[i] = bal[i-1] - delta[i-1]
前一行的借贷方向决定了到当前行的余额变化
```

**修正逻辑**：

- 余额差在容忍值内（默认 0.01 元）→ 标记"正确"
- 翻转借贷方向后吻合 → 自动修正并标记"已修正"
- 无法修正 → 标记"存疑"供人工复核

### 2.4 字段映射与标准化

`transaction_mapper.py` 实现了完整的字段标准化流程，支持 23 个标准字段：

**目标字段清单**：

| 字段分类 | 字段名 | 说明 |
|---------|--------|------|
| 标识字段 | id, serial_id | 序号标识 |
| 账户信息 | name, account_id, account_no, account_bank | 账户基本信息 |
| 时间信息 | trans_date, trans_time | 交易日期与时间 |
| 对方信息 | opponent_name, opponent_account_no, opponent_account_bank | 交易对手信息 |
| 金额信息 | debit_credit, trans_amt, account_balance, currency | 金额与余额 |
| 业务信息 | trans_channel, trans_type, trans_use, abstract, remark | 渠道与类型 |
| 校验信息 | verif_label, rel_line_num, create_time, update_time | 元数据字段 |

**映射策略**：

- 精确匹配字段名（如"对方户名"→"opponent_name"）
- 模糊匹配包含关系（如含"余额"→"account_balance"）
- 支持中英文别名（如"户名"→"name"，"支出"→"trans_amt"）
- 日期时间格式自动标准化（YYYY-MM-DD、HH:MM:SS）
- 借贷标志解析（借/贷/D/C/+/－等）

### 2.5 提示词动态管理

**Dify 集成架构**：

```
首次调用 → 请求 Dify API → 缓存 10 分钟
缓存未过期 → 直接返回缓存
Dify 不可用 → 优先使用旧缓存（即使过期）
完全无缓存 → 使用硬编码兜底提示词
```

**兜底提示词策略**：内置了完整的提示词模板，确保在 Dify 不可用时服务仍能正常运行。

## 三、API 接口设计

### 3.1 核心 OCR 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /ocr | POST | 异步受理，接收 URL 列表，立即返回 202，后台处理完成后回调 |
| /ocr/upload | POST | 同步处理，本地上传文件测试用，直接返回 CSV 结果 |

| 接口 | 说明 |

| POST /convert | PDF 转图片数组 |

| POST /convert/files | 单文件返回（JPEG） |
| POST /convert/image/{page} | 返回指定页码图片 |
| POST /convert/info | 获取 PDF 页数信息 |

### 3.3 字段映射服务

| 接口 | 说明 |
|------|------|
| POST /map | CSV 字段映射，返回标准化 CSV |
| POST /map/json | 返回 JSON 格式预览（限 100 条） |
| POST /map/info | 分析映射关系 |

## 四、技术亮点与创新

### 4.1 企业级可靠性设计

- **全局异常处理**：统一捕获未处理异常，返回标准错误响应
- **超时控制**：LLM 请求 1200 秒超时，HTTP 回调 60 秒
- **重试机制**：指数退避重试（1s→3s→5s），避免雪崩
- **回调保活**：连接复用关闭，防止 F5 断连
- **原始报文日志**：`RawLoggingMiddleware` 记录上下游请求/响应
- **Base64 掩码**：自动截断 base64 data URL，避免日志暴增

### 4.2 高并发优化

- **异步架构**：全程 async/await，无阻塞 IO
- **信号量控制**：`MAX_CHUNK_WORKERS` 限制 LLM 并发压力
- **PDF 分块**：大文件自动拆分并发处理
- **多文件并发**：PDF 与图片文件并行处理

### 4.3 数据质量保障

- **余额连续性校验**：基于财务规则的自动纠错
- **行数阈值检测**：防止 API 截断导致数据丢失
- **指纹去重**：多 chunk 合并时消除重复行
- **日期时间校验**：格式校验 + 日历合法性校验

### 4.4 可观测性

- **结构化日志**：带时间戳的 DEBUG/INFO/WARNING/ERROR 分级日志
- **缓存状态监控**：`cache_status()` API 暴露缓存健康状态
- **TTL 缓存**：提示词缓存 10 分钟，支持手动失效

| 配置项 | 说明 |
|--------|------|
| LLM_API_KEY | 阿里云百炼 API 密钥 |
| LLM_BASE_URL | API 基础 URL |
| LLM_MODEL | 模型选择（如 Qwen3-VL-Flash） |
| LLM_ENABLE_THINKING | 是否启用思考模式 |
| LLM_EXTRACT_TEMPERATURE | 提取温度参数 |

### 并发控制

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| MAX_WORKERS | 10 | 最大工作线程数 |
| CHUNK_SIZE | 8 | PDF 分块大小（页） |
| MAX_CHUNK_WORKERS | 6 | LLM 最大并发数 |

### PDF 处理

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| PDF_ZOOM | 2 | 缩放倍数 |
| PDF_QUALITY | 95 | 图片质量 |
| PDF_MAX_WIDTH | 1200 | 图片最大宽度 |
| IMAGE_FORMAT | JPEG | 输出格式 |

### 业务参数

| 配置项 | 默认值 | 说明 |
|--------|--------|------|

## 六、适用场景

### 6.1 银行流水数字化

将纸质/PDF 流水快速转为结构化数据，替代人工录入，大幅提升处理效率。

### 6.2 财务审计自动化

批量处理企业银行流水，支持余额校验与异常检测，辅助审计人员快速发现问题。
