
# PresentationAgent 技术分享报告

## 一、项目概述

### 1.1 项目定位
PresentationAgent 是一个基于 AI Agent 的智能演示文稿生成平台，通过 AI 技术自动化演示文稿的创建过程。用户只需提供主题或内容需求，系统即可自动完成从内容研究、逻辑组织到视觉呈现的全流程工作。

### 1.2 技术栈概览
- Frontend: Next.js 14 + React 18 + TypeScript + Tailwind CSS
- Backend: FastAPI + Python 异步 AIO
- LLM Layer: litellm 多模型客户端
- Runtime: AgentFactory + AgentRunner + MiddlewareChain
- Native Renderer: PPTXGENJS

---

## 二、核心架构（AgentFactory + AgentRunner + MiddlewareChain）

### 2.1 AgentFactory (Agent 工厂)
文件：backend/app/core/agent_factory.py
职责：动态构建 AgentContext + MiddlewareChain
关键方法：
- create() - 创建 AgentContext 和 MiddlewareChain
- _build_chain() - 根据意图组装中间件链
- create_subagent() - 创建子 Agent
设计模式：按意图动态选择中间件、动态组装工具集、定制系统提示词

### 2.2 AgentRunner (Agent 运行器)
文件：backend/app/core/agent_runner.py
职责：执行 Agent 主循环
核心流程：消息持久化 → on_request_start → 组装上下文 → 主循环 → on_request_end

### 2.3 MiddlewareChain (中间件链)
文件：backend/app/core/agent_middleware.py
设计模式：洋葱模型（正序执行 before，反序执行 after）
关键钩子：
- on_request_start: 请求开始前
- on_before_llm: 每轮 LLM 调用前
- on_after_llm: 每轮 LLM 响应后
- on_tool_start: 每次工具调用前
- on_tool_end: 每次工具调用后
- on_round_end: 每轮结束后
- on_request_end: 请求完全结束后

### 2.4 工具分发机制 (Tool Dispatch)
文件：backend/app/core/tool_dispatch.py
- dispatch() 函数执行工具
- filter_tools_by_intent() 按意图过滤工具
- load_skill() 动态加载 Skill
工具分类：RESEARCH, PPT, CODE_ANALYSIS, MEMORY, MEDIA, UTILITY, UNIVERSAL

### 2.5 上下文管理
文件：backend/app/services/context_service.py
- assemble_context() 组装上下文
- compress_context() 压缩机制处理长对话
- MODEL_CONTEXT_WINDOW 默认 128K tokens
- 智能截断：按工具类型区分保留长度

## 四、智能工作区

### 4.1 WebDeck 生成（PPT 演示文稿）
- 自研 Native Renderer PPTX 生成器
- WebSocket 实时通信
- 页面状态管理
- 支持主题定制：tech_dark, ocean_gradient, warm_sunset, forest_green, royal_purple, minimal_gray, coral_energy, classic_blue
- Director-Planner-Scheduler-Reviewer 四角色协同架构
- 支持导出为可编辑 PPTX 格式

核心工具：
- generate_ppt_deck: 生成完整 PPT
- edit_deck_page: 修改指定页面
- retry_failed_deck_pages: 重试失败页面

### 4.2 Drawio 流程图
- 生成 draw.io 兼容的流程图/架构图
- 支持 flowchart、UML 图、数据流图
- XML 格式输出
- 支持自定义样式和布局

### 4.3 Web 页面生成
- HTML 网页生成能力
- 支持可视化页面生成
- 响应式布局支持

### 4.4 智能文档
- Markdown 文档生成
- 研究报告输出
- 技术文档撰写

## 五、Docker 三层微服务架构

### 5.1 服务架构
- Backend (FastAPI + SQLAlchemy + 异步AIO)
- Frontend (Next.js 14 + React + TypeScript + Tailwind)
- Native Renderer (PPTXGENJS 渲染引擎)

### 5.2 通信机制
- 核心通信协议：WebSocket
- 二进制 PPTX 通过 Base64 传输
- 支持流式输出和实时进度反馈

### 5.3 前端状态管理
- Zustand 状态管理
- ChatStore 管理对话历史和流式内容
- DeckStore 管理 WebDeck 项目状态
- useWebSocket hook 实现全局单例 WebSocket
