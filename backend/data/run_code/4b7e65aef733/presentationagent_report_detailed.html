📋 一、项目概述

1.1 项目定位与背景

PresentationAgent 是一个基于 AI Agent 的智能演示文稿生成平台，通过 AI 技术自动化演示文稿的创建过程。用户只需提供主题或内容需求，系统即可自动完成从内容研究、逻辑组织到视觉呈现的全流程工作。

该项目的设计理念源自对现有 AI 辅助演示文稿生成工具的深度调研，核心目标是解决传统 PPT 制作过程中耗时费力的内容策划、视觉设计和排版优化等问题。通过引入自主研发的 Agent 运行时系统，实现了从自然语言指令到高质量演示文稿的端到端自动化。

🏗️ 二、核心架构详解

2.1 AgentFactory（Agent 工厂）

设计目标

AgentFactory 是整个运行时系统的核心入口，其设计目标是根据用户请求动态构建 Agent 实例。每个 Agent 实例都携带：

  • ● 动态组装的工具集：按意图过滤后的可用工具
  • ● 绑定的中间件链：不同任务类型绑定不同的中间件组合
  • ● 定制的系统提示词：按意图裁剪的 prompt sections
🔧

create()

创建 AgentContext 和 MiddlewareChain

⚙️

_build_chain()

根据意图组装中间件链

🔄

create_subagent()

创建子 Agent

2.2 AgentRunner（Agent 运行器）

核心流程
消息持久化 → on_request_start → 组装上下文 → 主循环 → on_request_end
// 主循环伪代码
async def _main_loop():
    while True:
        # 1. 组装当前上下文
        messages = await assemble_context(ctx)
        
        # 2. 调用 LLM 流式生成
        async for chunk in llm_stream(messages):
            yield chunk
        
        # 3. 解析 LLM 响应
        response = parse_response(chunks)
        
        # 4. 如果是工具调用，执行工具
        if response.has_tool_calls():
            for tool in response.tool_calls:
                result = await dispatch(tool)
                messages.append(result)
        
        # 5. 检查是否结束
        if response.is_final() or reached_max_rounds():
            break

2.3 MiddlewareChain（中间件链）

🧅 洋葱模型设计

中间件链的设计借鉴了 Express.js 中间件模式和 deer-flow 的 14 层有序中间件链设计，采用洋葱模型执行顺序：

正序执行 (before)
→
反序执行 (after)
钩子方法 触发时机 说明
on_request_start请求开始前初始化上下文
on_before_llm每轮 LLM 调用前可修改 messages
on_after_llm每轮 LLM 响应后后处理响应
on_tool_start工具调用前参数验证
on_tool_end工具调用后结果处理
on_round_end每轮结束后状态更新
on_request_end请求完全结束清理资源
